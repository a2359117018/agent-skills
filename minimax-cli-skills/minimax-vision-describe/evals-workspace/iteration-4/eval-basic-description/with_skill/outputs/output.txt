### 1. Special Functions (特有功能)
These are methods specifically available to the `List` interface beyond what is in the generic `Collection` interface, mostly revolving around index-based operations:
*   `void add(int index, Object obj)`: Inserts a specified element at a specified position.
*   `Object remove(int index)`: Removes the element at a specified index and returns it.
*   `Object set(int index, Object obj)`: Replaces the element at a specified index with a new value and returns the old value.
*   `int indexOf(Object o)`: Returns the index of the first occurrence of a specified element.
*   `Object get(int index)`: Retrieves the element at a specified position.
*   `ListIterator listIterator()`: Returns a list iterator for navigating the list in either direction.
*   `List subList(int fromIndex, int toIndex)`: Returns a portion of the list between the specified indices.

### 2. Classifications / Implementations (分类)
The mind map details three major implementations of the `List` interface:

#### **ArrayList**
*   **Characteristics (特点):**
    *   Uses an **array** as the underlying data structure.
    *   **Performance:** Fast for searching/accessing (queries) but slow for adding or removing elements (since it may require shifting elements).
    *   **Concurrency:** Not thread-safe, which results in higher efficiency.
*   **Special Functions:** It doesn't have unique methods of its own, but it is the most commonly used implementation.

#### **Vector**
*   **Characteristics (特点):**
    *   Also uses an **array** as the underlying data structure (fast queries, slow add/remove).
    *   **Concurrency:** It is **thread-safe**, which makes it slower (lower efficiency) compared to `ArrayList`.
*   **Legacy Special Functions (特有功能):**
    *   `addElement(E obj)`: Equivalent to `add()`.
    *   `elementAt(int index)`: Equivalent to `get()`.
    *   `elements()`: Equivalent to `iterator()`.

#### **LinkedList**
*   **Characteristics (特点):**
    *   Uses a **linked list** (doubly linked list) as the underlying data structure.
    *   **Performance:** Slow for searching/accessing by index but fast for adding or removing elements (no shifting needed).
    *   **Concurrency:** Not thread-safe, high efficiency.
*   **Special Functions (特有功能):** It includes specific methods for operating on the ends of the list:
    *   **Add:** `addFirst()`, `addLast()`
    *   **Remove:** `removeFirst()`, `removeLast()`
    *   **Get:** `getFirst()`, `getLast()`
