### **1. List 接口特有功能 (常用方法)**
List 集合是有序的，且带有索引，因此它具有一些基于索引操作的特有方法：
*   **`void add(int index, Object obj)`**: 在指定的索引位置添加元素。
*   **`Object remove(int index)`**: 根据指定索引删除元素，并返回被删除的元素。
*   **`Object set(int index, Object obj)`**: 修改指定索引位置的元素为新值，并返回修改前的值。
*   **`int indexOf(Object o)`**: 返回指定元素在集合中第一次出现的索引。
*   **`Object get(int index)`**: 获取指定索引位置的元素。
*   **`ListIterator listIterator()`**: 获取列表迭代器（List 专有的迭代器）。
*   **`List subList(int fromIndex, int toIndex)`**: 截取集合（获取子列表）。

### **2. 分类 (常见实现类)**

#### **ArrayList**
*   **特点**：
    *   底层数据结构是**数组**。
    *   查询速度快，增删速度慢。
    *   **线程不安全**，执行效率高。
*   **特有功能**：没有特有功能，但它是开发中最常用的实现类。

#### **Vector** (古老的实现类)
*   **特点**：
    *   底层数据结构也是**数组**。
    *   查询速度快，增删速度慢。
    *   **线程安全**，但执行效率低（现已较少使用）。
*   **特有功能**（旧式方法，现代开发建议使用标准接口方法）：
    *   `public void addElement(E obj)`: 相当于 `add()`。
    *   `public E elementAt(int index)`: 相当于 `get()`。
    *   `public Enumeration<E> elements()`: 相当于 `iterator()`。

#### **LinkedList**
*   **特点**：
    *   底层数据结构是**双向链表**。
    *   查询速度慢，增删速度快。
    *   **线程不安全**，执行效率高。
*   **特有功能**（主要针对首尾元素的操作）：
    *   **添加**：`addFirst()`、`addLast()`
    *   **删除**：`removeFirst()`、`removeLast()`
    *   **获取**：`getFirst()`、`getLast()`
