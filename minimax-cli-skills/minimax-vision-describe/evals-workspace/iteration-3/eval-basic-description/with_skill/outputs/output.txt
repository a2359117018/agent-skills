### 1. `List` Interface: Specific Functions (`特有功能`)
These are methods common to classes that implement the `List` interface. Unlike basic collections, `List` is index-based.

*   **`void add(int index, Object obj)`**: Adds an element at a specified position.
*   **`Object remove(int index)`**: Removes the element at the specified index and returns it.
*   **`Object set(int index, Object obj)`**: Replaces the element at a specified index with a new value and returns the old value.
*   **`int indexOf(Object o)`**: Returns the index of the first occurrence of the specified element.
*   **`Object get(int index)`**: Retrieves the element at the specified position.
*   **`ListIterator listIterator()`**: Returns a specialized iterator for lists that allows bidirectional traversal.
*   **`List subList(int fromIndex, int toIndex)`**: Returns a view of a portion of the list between the specified indices.

### 2. Classifications / Implementations (`分类`)

The mind map breaks down the three primary implementations of the `List` interface: `ArrayList`, `Vector`, and `LinkedList`.

#### **A. ArrayList**
*   **Features (`特点`):**
    *   **Structure:** Uses an internal array data structure.
    *   **Performance:** Fast for searching/accessing elements (`get`), but slow for adding or removing elements (because other elements must be shifted).
    *   **Safety/Efficiency:** It is **not thread-safe**, which makes it highly efficient in single-threaded environments.
*   **Unique Functions (`特有功能`):** It doesn't have major unique methods, but it is the **most commonly used** implementation.

#### **B. Vector**
*   **Features (`特点`):**
    *   **Structure:** Also uses an internal array data structure.
    *   **Performance:** Fast for searching, slow for adding/removing.
    *   **Safety/Efficiency:** It is **thread-safe** (synchronized), which makes it less efficient than `ArrayList`. It is generally considered a legacy class.
*   **Unique Functions (`特有功能`):** Contains older method names that are equivalent to standard List methods:
    *   `addElement(E obj)` → Equivalent to `add()`
    *   `elementAt(int index)` → Equivalent to `get()`
    *   `elements()` → Equivalent to `iterator()`

#### **C. LinkedList**
*   **Features (`特点`):**
    *   **Structure:** Uses a linked list (double-linked list) data structure.
    *   **Performance:** Slow for searching/accessing (must traverse nodes), but very fast for adding or removing elements at the ends.
    *   **Safety/Efficiency:** It is **not thread-safe**, offering high efficiency for specific tasks.
*   **Unique Functions (`特有功能`):** Because it's a linked list, it provides methods to easily interact with the ends of the list:
    *   **Adding:** `addFirst()`, `addLast()`
    *   **Removing:** `removeFirst()`, `removeLast()`
    *   **Getting:** `getFirst()`, `getLast()`
