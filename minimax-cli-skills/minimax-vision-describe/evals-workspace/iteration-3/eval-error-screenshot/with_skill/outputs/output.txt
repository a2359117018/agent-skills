**English Translation (for clarification):**
"...virtual machine, but faced many difficulties due to differences in Method Area implementation between the two. Considering the future development of HotSpot, during the JDK 6 era, the HotSpot development team already had plans to abandon the Permanent Generation (PermGen) and gradually move towards using Native Memory to implement the Method Area [1]. By JDK 7, HotSpot had already moved out the string constant pool, static variables, etc., which were originally placed in PermGen. Finally, in JDK 8, the concept of Permanent Generation was completely abandoned and replaced by Metaspace, which is implemented in native memory just like JRockit and J9, moving all remaining contents of PermGen from JDK 7 (mainly type information) into Metaspace."
