### 3. Data Flow

1.  **Input:** A search begins with a **Key** (e.g., John Smith).
2.  **Hashing:** The **Hash Function** (red arrows) transforms the key into a numerical **Index** (152).
3.  **Indexing:** The system looks at that specific slot in the **Hash Table**.
4.  **Collision Handling:** If the slot is occupied by a different key (as seen with the red arrows both pointing to 152), the system moves to the next available slot (153) to store or find the secondary record.
5.  **Retrieval:** The system follows the **Pointer** from the hash table index to the corresponding **Data Record** to retrieve the phone number.
